# Plan: Speech Model picker (download / select / delete local Whisper variants)
_Locked via grill-with-docs — by Claude + Bruno. Terms per CONTEXT.md. Revised after Codex round 1._

## Goal
Settings gains a **Speech Model** list under the Transcription engine section (visible when the local engine is selected): each row shows a multilingual Whisper variant with its state — Active, Downloaded, or Not downloaded — plus size. Users download models on demand (with live progress), select any downloaded model as Active (engine reloads in the background; in-flight Dictations finish on the old model), and delete non-Active downloaded models to reclaim disk. Only multilingual (EN+PT) variants are offered, so the Language Pin and PT Dictations can never hit an English-only model.

## Approach

1. **Curated catalog** — `SpeechModelCatalog` (new small file, pure data + pure functions): static list of multilingual variants with display name, variant id, approx size label:
   - `openai_whisper-tiny` (~150 MB, fastest), `openai_whisper-base`, `openai_whisper-small`, `openai_whisper-medium`, `openai_whisper-large-v3_947MB` (most accurate), `openai_whisper-large-v3-v20240930_turbo_632MB`, `openai_whisper-large-v3_turbo_954MB` (current default).
   - NO English-only (.en) and NO distil-* variants (distil models are English-only). large-v2 variants deliberately excluded: v3 supersedes them at the same size class — documented in the catalog's comment.
   - Device support, no-hang contract: the section renders IMMEDIATELY using the synchronous local `recommendedModels()` fallback config; a background task (launched once per app run, ~5s timeout) fetches `WhisperKit.recommendedRemoteModels()` and, on success, tightens/updates the cached support set and re-renders. Offline = fallback labels stand. Settings rendering never awaits network.
2. **AppSettings**: `whisperModel: String` — `@Published`, UserDefaults `didSet` persist (cloudSTTBaseURL pattern). Init-time normalization is SYNCHRONOUS and support-independent: prefixless legacy values alias to catalog ids ("large-v3_turbo_954MB" → "openai_whisper-large-v3_turbo_954MB"); anything not aliasable → static default `openai_whisper-large-v3_turbo_954MB`. Device-appropriate defaulting happens LATER and elsewhere: when `SpeechModelStore` gains a support set, IF the user has never explicitly chosen a model AND the current default isn't supported, the store applies the best supported entry (preference order: large-v3_turbo_954MB → large-v3-v20240930_turbo_632MB → large-v3_947MB → medium → small → base → tiny). USER-INTENT FLAG via separate APIs, not onChange sniffing: `setWhisperModelFromUser(_:)` (the ONLY path the Settings row uses) sets `whisperModelChosen`; `applyAutomaticWhisperModel(_:)` (device-default correction) and reload-failure reverts go through a non-flagging setter. AppSettings init never depends on async state. Active-model-becomes-unsupported policy (remote refresh may tighten labels after selection): a loaded Active model REMAINS usable, its row flagged "unsupported on this Mac"; automatic migration away from it happens only when NOT user-chosen.
3. **WhisperKitEngine** — variant becomes instance state, reload is ATOMIC-SWAP not reset-then-load:
   - Replace `static modelVariant` with the settings value CAPTURED at load time: `loadModel()` reads `AppSettings.shared.whisperModel` once (MainActor hop, same pattern as the vocabulary read) and remembers which variant it loaded (`loadedVariant: String?`).
   - GUARDED KIT STATE: `whisperKit` + `loadedVariant` live in ONE lock-guarded state struct (`OSAllocatedUnfairLock` or a small actor — match the file's existing lock idiom); EVERY read, write, and swap goes through it; no bare property access anywhere. `isLoaded` is DERIVED from that same guarded state (`whisperKit != nil`) — the separate `loadedFlag` Bool is deleted (a divergent flag invites stale "loaded" status across failed loads/reloads).
   - Kit acquisition for transcription: ONE helper (`kitForTranscription()`) reads the guarded state — kit present → return it (even mid-reload; reload never clears it); nil → run the normal gated `loadedKit()` load of the CURRENT setting. `performTranscribe` calls it once at entry and uses only that local reference throughout — an in-flight or queued transcription keeps the OLD kit even if a reload swaps the stored one mid-call.
   - `reload(to requested: String)`: guarded by its OWN reload lock (one reload at a time; a second request supersedes by re-checking the setting after finish). The new kit's download+load runs OUTSIDE the transcription gate — dictations keep transcribing on the old kit during a multi-minute model download; only the final swap of `whisperKit`/`loadedVariant` takes the tiny gated/locked critical section. On success: swap + status "Ready". On failure: old kit untouched, failure hint in status, and revert `AppSettings.whisperModel` to `loadedVariant` ONLY IF the setting still equals the failed `requested` variant (user may have already selected a third model — never clobber newer intent).
   - `isLoaded` caller audit: with `isLoaded` now derived from the guarded state ("some kit is loaded"; a reload swaps, never clears), AUDIT every caller (live-preview gating, Settings cloud-hint) against that definition and adjust any that assumed variant identity.
   - `name`/status strings include the active variant's display name (menu bar + Settings show which model is Ready).
4. **Model-state store — SINGLE observable source of truth, clear ownership boundary**: one `@MainActor ObservableObject` (`SpeechModelStore`) publishing per-variant `{phase: notDownloaded | downloading(percent) | downloaded | failed(hint) | busy(reloadTarget), sizeBytes?, active: Bool}`. Ownership split (exactly one layer calls `WhisperKit.download` per operation): the ENGINE performs load-time/reload downloads (its active variant) and EMITS events (variant, phase, percent, error) to the store via a narrow callback protocol; the STORE owns scans, sizes, deletes, and MANUAL row downloads. The engine's global `statusText` derives from the same events — row state and menu-bar status can't diverge. State publishing on MainActor; ALL filesystem work in background tasks with main-actor result hops. Single-download enforcement is CONCRETE, not asserted: a shared `SpeechModelDownloadCoordinator` actor is the only code that calls `WhisperKit.download` — engine load/reload downloads and store manual downloads both go through it; it holds one slot. Contention policy = REJECT-VISIBLY: while any download runs (either layer's), every other row's Download button is disabled with the caption "waiting for current download"; no hidden queue. A reload's TARGET variant is marked busy/non-deletable from reload start until success/failure — no deleting files under an in-progress load. Cancel: NOT offered (WhisperKit 0.18 exposes no cancellation handle; stuck download resolves by app restart — documented ponytail shortcut).
5. **Downloaded-state detection**: no WhisperKit list API — detection derives the variant's on-disk path from the SAME layout `WhisperKit.download` actually returns (HubApi cache layout; the build agent verifies against the really-downloaded model on this machine — a folder exists today — and encodes that resolution as `variantDirectory(for:baseURL:)`, a pure path function). "Downloaded" requires the expected CoreML artifacts present (at minimum the `*.mlmodelc` directories WhisperKit needs to load), NOT just a non-empty folder — partial/corrupt downloads classify as `.failed/.notDownloaded`, never `.downloaded`. Size = recursive folder size, computed off-main, cached per refresh. Rescan on section appear and after each download/delete.
6. **Delete**: per-row Delete button for downloaded, non-Active variants (Active row never shows delete). `confirmationDialog` with the size; deletion removes ONLY the exact resolved variant directory — never a repo/snapshot root, never siblings (other apps may share `~/Documents/huggingface`); path derived by the same `variantDirectory(for:)` resolution used by detection, with a defensive assertion validating the path against THAT function's output (not against the raw catalog id — resolved folder names may differ from catalog ids via WhisperKit's fuzzy matching). Deleting while mid-download is impossible (row shows progress, not delete).
7. **Settings UI** (SettingsView, Transcription engine section, `if settings.sttEngine == .whisperKit` block after the status Text): "Speech model" subsection — one row per catalog entry: radio-style selection (enabled when `.downloaded`), name + size label, trailing button by state (Download / progress% / Delete), unsupported rows disabled with caption. Selecting calls `setWhisperModelFromUser(_:)` — the SINGLE API that persists the value, sets the user-intent flag, and triggers the reload (Settings never writes `settings.whisperModel` directly). While a reload is already running, the newly selected variant renders "selected — pending reload" (row state, not just setting drift) until the superseding reload picks it up after the current one finishes. FIRST-LAUNCH state: the Active model may not be downloaded yet — engine preload emits `.downloading(percent)` through the store the moment it starts (identical row presentation to a manual download); before preload begins the row reads "Active — downloads on first use". It is Active and undeletable regardless of phase. The selectable-requires-downloaded rule applies to CHANGING selection, not to displaying the current one.
8. **Menu bar**: engine status line already flows from `statusText` — gains the model name for free via step 3. No new menu section.
9. **SelfCheck** (no network, no real model downloads; fakes/temp dirs allowed):
   - Catalog invariants: all entries multilingual (no ".en", no "distil"), unique ids, preference order covers every entry.
   - Settings normalization: prefixless alias maps to catalog id; unknown → device default; catalog variant kept. Device-default resolution over fabricated support sets (incl. empty/unknown).
   - `variantDirectory(for:)` path mapping: pure checks over fabricated base URLs; resolver self-consistency (detect and delete agree on the same path) and containment under the expected repo cache root (never resolves to the root itself or outside it).
   - Downloaded-detection over fabricated temp directory trees: missing dir → notDownloaded; dir with junk but no .mlmodelc → NOT downloaded; expected artifacts → downloaded.
   - Reload decision (pure fn): loadedVariant nil/equal → no reload; differing → reload.
   - Reload FAILURE preservation via injected fake loader: failed new-kit load keeps the old kit usable and reverts the setting; concurrent-transcribe kit-capture (fake kit identity check: transcription started before swap reports the OLD kit's identity).
   - Guarded kit-state transitions (fake kit objects): nil → loaded; reload swap replaces identity; failed reload preserves old identity; `isLoaded == (kit != nil)` across all of the above.
   - Delete eligibility (pure fn): active → never deletable (even when notDownloaded); downloading → not deletable; downloaded non-active → deletable.
   - Mutation-test each (invert → selfcheck FAILS → restore).
10. **README**: Speech model section (what the variants trade off, where they're stored, delete). Plus one extra note (from grill): local post-processing LLMs already work via Ollama's desktop app + the BYOK provider at `http://localhost:11434/v1` with no key — document, zero code.

## Key decisions & tradeoffs
- **STT models only** — local post-processing LLMs rejected: needs llama.cpp/MLX runtime (huge new dependency) while local Ollama already covers it through the existing OpenAI-compatible BYOK path (README note instead).
- **Multilingual-only curated catalog** — .en/distil variants would silently break PT dictations and the Language Pin; a curated static list beats live `fetchAvailableModels` (network, unfiltered, includes unsupported/duplicate variants). large-v2 excluded on purpose (v3 supersedes at same size class).
- **Reload = load-new-then-swap, never clear-then-load** — a failed download/load keeps the previously working model; the selection reverts visibly.
- **In-flight old-model guarantee = kit capture inside performTranscribe** — not the coordinator's engine snapshot (single shared engine makes that insufficient).
- **Support set from recommendedRemoteModels with local fallback** — remote is current, local fallback config is the no-network answer; cached per run.
- **Download and select are separate** — no surprise multi-GB downloads; only downloaded models selectable; mid-download switching impossible by construction.
- **No download cancellation** — WhisperKit 0.18 has no cancel API; marked ponytail shortcut, restart clears a stuck download.
- **Active model undeletable; reload target busy-locked** — no "deleted the model under the engine/loader" state.

## Risks / open questions
- Variant folder naming on disk must match catalog ids (WhisperKit's fuzzy matching may create differently-prefixed folders) — build agent verifies against the actual download-base layout on this machine (a model is already downloaded) and adjusts the detection glob accordingly.
- `~/Documents/huggingface` may require Documents-folder TCC prompt on first scan in some configurations — accepted; the app already downloads there today.
- Medium/large downloads on slow networks hold the single download slot; acceptable for a personal tool.

## Out of scope
- Local post-processing LLM runtime (Ollama README note only).
- English-only/distil variants; custom HF repos; token-gated models.
- Download cancellation/pause/resume; parallel downloads.
- Changing the storage location (`downloadBase` stays WhisperKit's default).
